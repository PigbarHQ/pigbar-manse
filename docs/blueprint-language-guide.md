# Blueprint Language Guide

작성일: 2026-07-03

## 1. 목적

Blueprint의 언어는 제품 철학을 지키는 첫 번째 인터페이스다.

Blueprint는 사주 리포트 서비스가 아니다. Blueprint는 한 사람을 한 권의 책으로 출판하고, 그 책들이 모여 도서관이 되는 서비스다. 따라서 제품 전체의 단어는 리포트, 분석, 결과, 공유, 팔로우 같은 일반 SaaS/SNS 언어가 아니라 책, 출판, 서재, 도서관, 사서, 개정판의 언어를 따라야 한다.

이 문서는 UI, 문서, API display copy, 결제 문구, 알림 문구, 고객 응대에서 사용할 표준 용어와 금지 용어를 정의한다.

## 2. 핵심 언어 원칙

- 사람을 평가하지 않는다. 읽는다.
- 결과를 생성하지 않는다. 책을 출판한다.
- 사람을 공유하지 않는다. 책을 추천한다.
- 데이터를 삭제하지 않는다. 책을 절판하거나 서가에서 내린다.
- 프로필을 만들지 않는다. 저자 소개를 둔다.
- 피드를 만들지 않는다. 도서관과 서가를 만든다.
- AI를 앞세우지 않는다. 사서와 편집자의 역할로 설명한다.
- 업데이트하지 않는다. 개정판을 발행한다.

## 3. 표준 용어 변환표

| 일반 서비스 용어 | Blueprint 표준 용어 | 사용 맥락 | 비고 |
|---|---|---|---|
| 리포트 | 책 | 사용자 결과물 전체 | "사주 리포트" 금지 |
| 결과 | 책 / 원고 / 본문 | 계산 후 사용자에게 보이는 산출물 | 상황에 따라 구분 |
| 생성 | 출판 | 첫 책을 만드는 행위 | "책 출판하기" |
| 다시 생성 | 개정판 발행 | 기존 책을 새 버전으로 갱신 | 무작위 재생성 느낌 금지 |
| 삭제 | 절판 | 책 자체를 더 이상 제공하지 않음 | 완전 삭제는 내부 정책명으로만 |
| 숨기기 | 서가에서 내리기 | 공개/노출 중지 | Library 문맥 |
| 저장 | 내 서재에 꽂기 / 보관 | 개인 보관 | 짧은 UI에서는 "보관" 허용 |
| 즐겨찾기 | 내 서재 | 개인 보관 공간 | 기능명으로 "즐겨찾기" 금지 |
| 공유 | 추천 | 다른 사람에게 책을 권함 | SNS 공유 뉘앙스 금지 |
| 링크 공유 | 추천 링크 | 책 추천용 링크 | "공유 링크"보다 우선 |
| 프로필 | 저자 소개 | 사용자/책 소유자 소개 | 사람보다 책이 앞 |
| 유저 | 독자 / 저자 | 읽을 때는 독자, 책 주인은 저자 | 관리자 문서에서도 가능하면 구분 |
| 회원 | 도서관 회원 | 멤버십 문맥 | 결제/구독에 사용 |
| 팔로우 | 사용 금지 | - | 사람을 따르지 않음 |
| 팔로워 | 사용 금지 | - | 숫자 경쟁 금지 |
| 좋아요 | 사용 금지 | - | 반응 수 금지 |
| 댓글 | 사용 금지 | - | 공개 평가 금지 |
| 피드 | 사용 금지 / 전시 | 공개 탐색은 전시/서가 | 무한 스크롤 금지 |
| 라이브러리 | 도서관 | 제품의 공개/공동 책 공간 | 영어 브랜드 영역 제외 |
| 검색 | 서가 탐색 | 도서관 탐색 | 기술 문서에서는 search 허용 |
| 카테고리 | 서가 | 책 분류 | "관계 서가", "성장 서가" |
| 추천 | 사서 추천 / 책 추천 | 큐레이션 | 사람 추천 금지 |
| 인기 | 많이 읽힌 책 | 공개 지표 최소화 | 랭킹화 금지 |
| 랭킹 | 사용 금지 | - | 경쟁 구조 금지 |
| 알림 | 소식 | 조용한 안내 | 숫자 배지 금지 |
| AI | 사서 / 편집자 | 사용자-facing 표현 | 기술 문서에서는 AI 가능 |
| 챗봇 | 사서 | 도움/탐색 역할 | "사서에게 묻기" |
| 프롬프트 | 편집 지시 / 원고 지시 | 내부 문서 | 사용자-facing 금지 |
| 버전 | 초판 / 개정판 | 책의 변경 이력 | "v2"보다 "2판" |
| 업데이트 | 개정판 발행 | 책 내용 갱신 | "업데이트 완료" 금지 |
| 히스토리 | 판본 기록 | 개정 이력 | "개정 기록"도 가능 |
| 대시보드 | 서재 / 작업실 | 사용자 공간 | 운영자 화면은 dashboard 허용 |
| 홈 | 내 서재 | 로그인 후 첫 화면 | 마케팅 사이트는 홈 가능 |
| 분석 | 구조 읽기 | 사람 설명 | "성격 분석" 금지 |
| 점수 | 지표 / 경향 / 강도 | 내부 feature 표시 | 사용자에게 점수 남발 금지 |
| 진단 | 구조 해석 | 건강/심리 진단 오해 방지 | 의료적 뉘앙스 금지 |
| 운세 | 사용 금지 | - | Blueprint는 운세 서비스 아님 |
| 예측 | 흐름 읽기 | 미래 문맥 | 단정 예측 금지 |
| 결제 | 도서관 회원권 / 책 출판 비용 | 결제 UX | 법적 표시에는 결제 병기 가능 |
| 구독 | 도서관 이용권 | 멤버십 | 구독 강요 금지 |
| 플랜 | 이용권 | 요금제 문맥 | "월 이용권", "연 이용권" |
| 고객센터 | 안내 데스크 | 도움말 문맥 | 법적 페이지는 고객센터 허용 |
| 문의 | 사서에게 문의 | 도움 요청 | 차분한 톤 |

## 4. 반드시 사용할 단어

Blueprint의 핵심 화면과 문서에서 우선 사용한다.

| 표준 단어 | 의미 | 예문 |
|---|---|---|
| 책 | 한 사람의 Blueprint 결과물 | "내 책을 읽어보세요." |
| 원고 | 출판 전 초안 | "원고를 다듬고 있습니다." |
| 출판 | 책을 처음 만드는 행위 | "내 책 출판하기" |
| 초판 | 첫 번째 책 | "초판이 준비되었습니다." |
| 개정판 | 다시 다듬은 책 | "새 개정판을 발행했습니다." |
| 절판 | 책을 더 이상 제공하지 않음 | "이 책을 절판하시겠습니까?" |
| 내 서재 | 개인 보관 공간 | "내 서재에 꽂기" |
| 도서관 | 책들이 모이는 공간 | "Blueprint 도서관 둘러보기" |
| 서가 | 책의 분류/탐색 단위 | "관계 서가" |
| 전시 | 큐레이션된 책 묶음 | "이번 달 전시" |
| 사서 | 안내/추천/탐색 역할 | "사서 추천" |
| 편집자 | 문장 품질 관리 역할 | "편집자가 원고를 점검했습니다." |
| 저자 | 자기 책의 주인 | "저자 소개" |
| 독자 | 책을 읽는 사람 | "독자 메모" |
| 책갈피 | 읽던 위치 저장 | "책갈피를 남겼습니다." |
| 밑줄 | 중요한 문장 표시 | "밑줄 친 문장" |
| 메모 | 개인 기록 | "내 메모" |
| 이어읽기 | 읽던 곳부터 다시 읽기 | "이어서 읽기" |
| 판본 | 책의 발행 이력 | "판본 기록" |
| 소장 | 내 책을 오래 보관함 | "평생 소장" |

## 5. 절대 사용하지 않을 단어

사용자-facing UI, 마케팅, Reader, Library, 결제 유도 문구에서 금지한다.

| 금지 단어 | 금지 이유 | 대체어 |
|---|---|---|
| 리포트 | Blueprint를 일반 분석물로 축소함 | 책 |
| 운세 | 점술/일회성 소비로 오해됨 | 흐름 / 구조 |
| 점괘 | 공포/예언 뉘앙스 | 사용 금지 |
| 예언 | 미래 단정 | 흐름 읽기 |
| 적중 | 점술 마케팅 뉘앙스 | 사용 금지 |
| 대박운 | 자극적 운세 장사 | 확장 국면 |
| 흉운 | 공포 유도 | 주의 구간 / 압박 구간 |
| 팔자 | 체념/운명론 뉘앙스 | 구조 |
| 숙명 | 닫힌 운명론 | 구조 / 경향 |
| 등급 | 사람 평가 | 경향 / 강도 |
| 랭킹 | 경쟁과 비교 유도 | 전시 / 추천 |
| 좋아요 | 반응 수 경쟁 | 밑줄 / 추천 |
| 댓글 | 공개 평가와 논쟁 유도 | 개인 메모 |
| 팔로워 | 영향력 게임 | 독자 / 도서관 회원 |
| 피드 | 소비형 UX | 도서관 / 전시 |
| 바이럴 | 성장 지표 중심 | 추천 |
| 중독 | 제품 철학 위반 | 몰입 / 독서 |
| 프로필 | 사람을 전시물로 만듦 | 저자 소개 |
| 유저 | 사람을 기능 단위로 축소 | 독자 / 저자 |
| AI 생성 | 기계가 책을 만든다는 인상 | 편집 / 원고 정리 |
| 삭제 | 소장 경험을 해침 | 절판 / 서가에서 내리기 |
| 재생성 | 무작위 결과물 느낌 | 개정판 발행 |
| 분석 결과 | 리포트 느낌 | 책 / 구조 읽기 |
| 성격 분석 | 사람 단정 | 구조 읽기 |

## 6. 기능별 표준 문구

### 6.1 책 출판

사용:

- 내 책 출판하기
- 초판 만들기
- 원고 준비 중
- 책이 준비되었습니다

금지:

- 리포트 생성하기
- 결과 생성 중
- AI가 분석 중

### 6.2 Reader

사용:

- 이어서 읽기
- 책갈피 남기기
- 밑줄 긋기
- 내 메모 쓰기
- 내 결론 남기기

금지:

- 결과 보기
- 분석 확인
- 좋아요
- 댓글 쓰기

### 6.3 Library

사용:

- 도서관 둘러보기
- 서가 탐색
- 사서 추천
- 오늘의 한 권
- 이번 달 전시

금지:

- 피드 보기
- 인기 랭킹
- 팔로우하기
- 유저 탐색

### 6.4 개정판

사용:

- 개정판 발행
- 판본 기록
- 초판으로 돌아가기
- 새 판을 준비 중입니다

금지:

- 다시 생성
- 업데이트
- 리셋
- 버전 업데이트

### 6.5 결제/멤버십

사용:

- 책 출판 비용
- 도서관 이용권
- 도서관 회원
- 내 책은 평생 소장됩니다

금지:

- 운세 구매
- 리포트 구매
- 구독하지 않으면 사라집니다
- 지금 결제하지 않으면 손해입니다

## 7. 톤 가이드

Blueprint의 문장은 조용하고 단단해야 한다.

사용할 톤:

- 침착한
- 정확한
- 품격 있는
- 오래 읽히는
- 단정하지 않는
- 과장하지 않는

피할 톤:

- 겁주는
- 과하게 다정한
- 점술가 같은
- 마케팅 냄새가 강한
- 클릭을 부르는
- 사람을 단정하는

좋은 문장:

```text
결정을 미루는 사람이 아니라, 기준이 충분히 설 때까지 움직이지 않는 사람입니다.
```

나쁜 문장:

```text
당신은 우유부단하지만 곧 좋은 운이 옵니다.
```

## 8. 예외 규칙

법적/운영 문서에서는 일부 일반 용어를 병기할 수 있다.

예:

- 결제
- 환불
- 계정
- 개인정보
- 고객센터
- 삭제

단, 사용자 경험의 중심 화면에서는 Blueprint 표준 용어를 우선한다.

예:

```text
도서관 이용권 결제
책 절판 및 데이터 삭제
안내 데스크 / 고객센터
```

## 9. 최종 원칙

Blueprint의 언어는 사용자를 빠르게 움직이게 하기보다 오래 읽게 해야 한다.

우리는 리포트를 만들지 않는다. 책을 출판한다.

우리는 사람을 공유하지 않는다. 책을 추천한다.

우리는 피드를 만들지 않는다. 도서관을 만든다.

우리는 AI를 앞세우지 않는다. 사서와 편집자의 책임을 앞세운다.
