# Blueprint Living Book

작성일: 2026-07-03

## 1. 목적

Blueprint Living Book은 Blueprint Book을 단순 PDF나 고정된 결과물이 아니라, 사용자가 평생 다시 읽고 대화할 수 있는 살아 있는 책으로 정의한다.

Living Book은 챗봇이 아니다. 운세 상담도 아니고, 사람을 대신 판단하는 도구도 아니다. Living Book은 이미 출판된 책을 기반으로 독자가 자신의 구조를 더 깊이 읽도록 돕는 독서 동반자다.

핵심 문장:

```text
책은 출판되지만, 독서는 계속된다.
Blueprint Book은 완성된 문서이면서 동시에 평생 대화할 수 있는 책이다.
Living Book은 사람을 대신 말하지 않고, 사람이 자신을 다시 읽도록 돕는다.
```

## 2. Living Book 개념

일반적인 PDF는 한 번 다운로드되고 끝난다. Blueprint Book은 다르다. 한 사람의 구조는 시간이 지나도 반복되는 축을 가지지만, 그 구조를 이해하는 독자의 시점은 계속 변한다.

Living Book은 다음을 가능하게 한다.

- 책 전체와 대화한다.
- 특정 챕터와 대화한다.
- 저자 자신과의 대화를 기록한다.
- 판본별로 달라진 해석을 비교한다.
- 시간이 지나며 메모, 밑줄, 질문, 회고가 쌓인다.
- 책이 독자의 삶을 대신 결론 내리지 않고, 다시 읽을 질문을 남긴다.

Living Book의 본질:

- 고정된 책
- 계속되는 독서
- 축적되는 대화
- 보존되는 판본
- 깊어지는 자기 이해

Living Book은 "새 답을 무한히 생성하는 AI"가 아니라 "한 권의 책을 계속 더 깊게 읽게 하는 Reader Experience"다.

## 3. Book Chat

Book Chat은 책 전체를 대상으로 하는 대화다.

사용자는 책을 다 읽은 뒤, 또는 읽는 중간에 책 전체에 질문할 수 있다. Book Chat은 책의 내용, Core Feature, Manse 계산 결과, 판본 기록, 사용자의 메모를 바탕으로 답한다.

### 3.1 Book Chat의 역할

- 책 전체의 핵심 구조를 다시 설명한다.
- 서로 다른 챕터 사이의 연결을 찾아준다.
- 독자가 밑줄 친 문장을 바탕으로 질문을 확장한다.
- 책의 주요 긴장과 반복 패턴을 정리한다.
- 독자가 자기 언어로 결론을 적을 수 있도록 돕는다.

### 3.2 가능한 질문

- 이 책에서 가장 중요한 문장은 무엇인가요?
- 내 결정 방식은 어느 챕터에서 가장 잘 드러나나요?
- 이 책이 반복해서 말하는 내 구조는 무엇인가요?
- 관계 챕터와 성장 챕터는 어떻게 연결되나요?
- 내가 밑줄 친 문장들을 하나의 문단으로 정리해줄 수 있나요?

### 3.3 Book Chat의 원칙

- 책에 없는 사건을 만들지 않는다.
- 독자에게 행동을 강요하지 않는다.
- 미래를 예언하지 않는다.
- 책을 요약하되 사람을 납작하게 만들지 않는다.
- 답변마다 근거가 되는 챕터 또는 문장을 표시한다.

## 4. Chapter Chat

Chapter Chat은 특정 챕터와 대화하는 기능이다.

Blueprint Book의 각 챕터는 하나의 질문을 가진다. Chapter Chat은 그 질문을 중심으로 독자가 더 오래 머물 수 있게 돕는다.

### 4.1 Chapter Chat의 역할

- 해당 챕터의 핵심 문장을 다시 읽어준다.
- 어려운 문장을 쉽게 풀어준다.
- 챕터 안의 긴장과 전환을 설명한다.
- 독자의 메모를 바탕으로 회고 질문을 제안한다.
- 다음 챕터와의 연결을 안내한다.

### 4.2 챕터별 대화 예시

Chapter 1. 나는 누구인가:

- 이 챕터가 말하는 내 중심 구조는 무엇인가요?
- 이 문장을 더 쉽게 설명해줄 수 있나요?
- 이 챕터에서 강점과 위험은 각각 무엇인가요?

Chapter 4. 나는 사람을 어떻게 대하는가:

- 이 챕터는 내 관계 방식을 어떻게 설명하나요?
- 가까운 관계에서 반복되는 패턴은 무엇인가요?
- 이 챕터의 마지막 문장은 무슨 뜻인가요?

Chapter 10. 지금 나는 어디에 있는가:

- 현재 흐름을 단정하지 않고 읽는다면 어떻게 볼 수 있나요?
- 지금 필요한 태도는 무엇인가요?
- 이 장에서 조심해야 할 과장은 무엇인가요?

### 4.3 Chapter Chat의 제한

- 챕터 범위를 벗어난 답변은 Book Chat으로 넘긴다.
- 의료, 법률, 투자, 심리 진단 질문에는 답하지 않는다.
- 특정 타인의 마음을 추측하지 않는다.
- 사용자에게 결정을 대신 내려주지 않는다.

## 5. Author Conversation

Author Conversation은 사용자가 자기 책의 저자로서 남기는 대화다.

Blueprint에서 저자는 AI가 아니다. 저자는 자기 책을 소유한 사람이다. AI는 사서 또는 편집자로서 독자가 자신을 더 잘 읽도록 돕는다.

### 5.1 Author Conversation의 목적

- 사용자가 책을 읽으며 느낀 자기 해석을 기록한다.
- 시간이 지난 뒤 같은 문장을 다시 읽고 달라진 생각을 남긴다.
- 개정판 발행 시 반영할 수 있는 개인 메모를 축적한다.
- 책이 독자에게 일방적으로 말하는 구조를 넘어서게 한다.

### 5.2 가능한 대화

- 이 문장이 지금은 다르게 느껴져요.
- 예전에는 이 부분이 불편했는데 이제 이해됩니다.
- 이 챕터에 내 경험을 메모로 남기고 싶어요.
- 다음 개정판에 반영하고 싶은 변화가 있어요.
- 이 표현은 나와 잘 맞지 않아요.

### 5.3 Author Conversation의 원칙

- 사용자의 말은 사용자의 기록으로 남긴다.
- AI가 사용자의 경험을 사실처럼 확장하지 않는다.
- 사용자의 메모와 책 본문을 구분한다.
- 개정판에 반영할 때는 반드시 변경 이력을 남긴다.
- 저자의 자기 해석을 존중하되, 계산 근거와 구분한다.

## 6. Memory Layer

Memory Layer는 Living Book의 장기 기억 계층이다.

Memory Layer는 사용자를 감시하기 위한 기능이 아니다. 독서 경험이 단절되지 않도록 책갈피, 밑줄, 메모, 질문, 판본 기록을 보존하는 구조다.

### 6.1 Memory Layer에 저장되는 것

- 책갈피
- 밑줄
- 개인 메모
- Book Chat 질문과 답변
- Chapter Chat 질문과 답변
- Author Conversation 기록
- 개정판 요청 사유
- 판본별 읽은 위치
- 독자가 직접 작성한 결론

### 6.2 Memory Layer에 저장하지 말아야 할 것

- 사용자의 민감한 사생활을 추측한 내용
- 사용자가 말하지 않은 경험
- 제3자의 개인정보
- 의료, 법률, 투자 판단
- AI가 만들어낸 사건
- 사용자의 불안을 자극하기 위한 태그

### 6.3 Memory Layer 원칙

- 사용자가 쓴 것과 AI가 답한 것을 분리한다.
- 원문, 요약, 해석을 분리한다.
- 삭제보다 절판과 보관의 언어를 쓰되, 개인정보 삭제 권리는 별도로 보장한다.
- 판본이 바뀌어도 이전 대화 맥락을 잃지 않는다.
- Memory는 독자를 더 잘 팔기 위한 데이터가 아니라, 독자가 자신을 더 잘 읽기 위한 기록이다.

## 7. Edition별 대화

Living Book은 판본별로 대화가 달라질 수 있다.

초판과 2판은 같은 사람의 책이지만 같은 책은 아니다. 문장, 구조, 계산 엔진, 사용자 메모, 개정 이유가 달라질 수 있다. 따라서 대화 역시 어떤 판본을 기준으로 답하는지 명확해야 한다.

### 7.1 Edition Context

모든 대화는 다음 판본 정보를 가져야 한다.

```json
{
  "blueprintId": "bp_...",
  "editionNumber": 2,
  "editionName": "2판",
  "publicationDate": "2027-01-10T00:00:00+09:00",
  "conversationScope": "book",
  "sourceEdition": "2판"
}
```

### 7.2 판본별 답변 원칙

- 현재 읽고 있는 판본을 기준으로 답한다.
- 이전 판본과 비교할 때는 차이를 명확히 표시한다.
- 초판의 문장을 2판의 문장처럼 인용하지 않는다.
- 개정판에서 사라진 문장은 "이전 판본의 문장"으로 표시한다.
- 계산 엔진 개선으로 달라진 부분과 문체 개선으로 달라진 부분을 구분한다.

### 7.3 판본별 대화 예시

사용자:

```text
초판과 2판에서 내가 달라진 부분은 뭐예요?
```

Living Book:

```text
2판에서 달라진 것은 당신의 사람이 바뀌었다는 뜻이 아닙니다.
책이 당신의 구조를 설명하는 방식이 더 선명해졌다는 뜻에 가깝습니다.

초판은 관계의 신중함을 약점처럼 설명했지만,
2판은 그것을 신뢰가 만들어지는 속도로 다시 정리했습니다.
```

## 8. AI가 답변할 수 있는 범위

Living Book의 AI는 사서 또는 편집자다. 답변 범위는 책과 근거가 있는 기록 안으로 제한된다.

### 8.1 답변 가능 범위

- 책 본문 설명
- 챕터 요약
- 문장 해석
- Core Feature 기반 구조 설명
- 밑줄과 메모 정리
- 판본별 차이 설명
- 독서 질문 제안
- 개정판에 반영할 메모 정리
- Reader UX 안내
- 도서관 안에서 책을 찾는 도움

### 8.2 답변할 때 필요한 근거

답변은 다음 중 하나 이상의 근거를 가져야 한다.

- 책 본문
- 챕터 제목
- Core Feature
- Manse 계산 결과
- 사용자의 명시적 메모
- 판본 기록
- Quality/Fact Check 기록

근거가 없으면 "이 책만으로는 말할 수 없습니다"라고 답해야 한다.

### 8.3 좋은 답변 태도

- 짧게 답하고, 필요하면 더 읽을 문장을 제안한다.
- 단정 대신 가능성을 말한다.
- 불안 대신 구조를 설명한다.
- 독자의 선택권을 남긴다.
- 책의 어느 부분에서 온 답인지 표시한다.

## 9. 답변하면 안 되는 범위

Living Book은 모든 질문에 답하지 않는다. 답하지 않는 능력은 Living Book의 품질이다.

### 9.1 금지 범위

- 미래 사건 예언
- 결혼, 이별, 합격, 당첨, 질병, 사고의 단정
- 의료 진단
- 법률 판단
- 투자 판단
- 심리 질환 진단
- 타인의 마음 추측
- 제3자의 사생활 해석
- 공포를 이용한 행동 유도
- 결제를 유도하기 위한 불안 조성
- 책에 없는 경험을 사실처럼 말하기
- 사용자의 삶을 대신 결정하기

### 9.2 거절 문장 원칙

거절은 차갑지 않아야 한다. 대신 경계를 분명히 해야 한다.

좋은 거절:

```text
이 책만으로 그 사건이 일어날지 단정할 수는 없습니다.
다만 이 책은 당신이 불확실한 상황에서 어떤 기준을 세우는지 읽을 수 있습니다.
```

나쁜 거절:

```text
답변할 수 없습니다.
정책상 불가능합니다.
그건 제 역할이 아닙니다.
```

### 9.3 위험 질문 전환

금지 질문은 안전한 독서 질문으로 전환한다.

| 위험 질문 | 전환 질문 |
|---|---|
| 올해 결혼할까요? | 가까운 관계에서 내가 반복하는 기준은 무엇인가요? |
| 이 사람은 나를 좋아하나요? | 나는 관계에서 어떤 신호를 신뢰하나요? |
| 투자해도 될까요? | 자원을 다룰 때 내가 조심해야 할 패턴은 무엇인가요? |
| 이직해야 하나요? | 변화 앞에서 내가 확인해야 할 기준은 무엇인가요? |
| 병이 생길까요? | 몸과 회복 리듬을 어떻게 살펴야 하나요? |

## 10. Reader UX

Living Book UX는 채팅 앱이 아니라 책 읽기의 확장이어야 한다.

### 10.1 기본 원칙

- 대화보다 책이 먼저 보인다.
- 채팅창이 독서를 침범하지 않는다.
- 질문은 문장 옆에서 조용히 열린다.
- 답변은 길게 쌓이지 않고 책갈피처럼 남는다.
- 대화 기록은 독자의 Memory Layer에 보관된다.
- 알림, 배지, 반응 수로 독서를 방해하지 않는다.

### 10.2 주요 UX

#### Book Chat

- 책 전체 하단 또는 사이드 패널에서 열린다.
- "이 책에게 묻기"처럼 표현한다.
- 답변에는 관련 챕터 링크가 붙는다.

#### Chapter Chat

- 각 챕터 끝에서 조용히 열린다.
- "이 장을 더 읽기"처럼 표현한다.
- 챕터 범위를 넘어가면 Book Chat으로 안내한다.

#### Sentence Ask

- 밑줄 친 문장 옆에서 열린다.
- "이 문장 다시 읽기"처럼 표현한다.
- 해당 문장을 쉬운 말로 풀거나, 앞뒤 문맥과 연결한다.

#### Author Notes

- 독자가 자기 문장을 남긴다.
- AI 답변과 시각적으로 구분한다.
- 개정판 반영 후보로 저장할 수 있다.

### 10.3 사용하지 않을 UX

- 무한 채팅 피드
- 타이핑 경쟁
- AI 캐릭터화
- 상담사처럼 행동하는 인터페이스
- 긴급 운세 알림
- 결제 유도 질문
- 좋아요, 댓글, 팔로워

## 11. Living Book Architecture

Living Book Architecture는 책, 대화, 기억, 판본을 분리한다.

```text
Published Book
↓
Reader Context
↓
Conversation Scope
↓
Memory Layer
↓
Answer Engine
↓
Safety Boundary
↓
Edition Archive
```

### 11.1 Core Components

| Component | 역할 |
|---|---|
| Published Book | 출판된 책 본문과 메타데이터 |
| Reader Context | 현재 읽는 위치, 밑줄, 책갈피, 메모 |
| Conversation Scope | Book, Chapter, Sentence, Edition 범위 |
| Memory Layer | 장기 독서 기록과 대화 기록 |
| Answer Engine | 근거 기반 답변 생성 |
| Safety Boundary | 답변 가능 범위와 금지 범위 검사 |
| Edition Archive | 판본별 본문, 대화, 변경 이력 보존 |

### 11.2 Conversation Request Schema

```json
{
  "blueprintId": "bp_...",
  "editionNumber": 1,
  "scope": {
    "type": "chapter",
    "chapterNo": 4,
    "paragraphId": "p_0042"
  },
```

### 11.4 Safety Boundary

모든 답변 전 다음을 확인한다.

- 질문이 책 범위 안에 있는가
- 답변 근거가 있는가
- 미래 사건을 단정하지 않는가
- 의료/법률/투자/심리 진단으로 넘어가지 않는가
- 제3자 정보를 추측하지 않는가
- 결제를 유도하기 위해 불안을 만들지 않는가

### 11.5 Memory와 Edition 분리

Memory는 사용자의 독서 기록이다. Edition은 책의 판본 기록이다.

- Memory는 사용자별로 쌓인다.
- Edition은 책별로 보존된다.
- 같은 책의 다른 독자는 다른 Memory를 가진다.
- 같은 사용자는 여러 Edition에 걸쳐 Memory를 가질 수 있다.
- 개정판이 나와도 이전 판본의 대화는 사라지지 않는다.

## 12. 장기 비전

Living Book의 장기 비전은 책이 사람과 함께 늙어가는 것이다.

Blueprint Book은 출판 순간에 끝나지 않는다. 독자가 20대에 읽은 문장과 40대에 다시 읽은 문장은 다르게 느껴질 수 있다. 같은 구조라도 삶의 시간에 따라 해석은 깊어진다.

장기적으로 Living Book은 다음을 지향한다.

### 12.1 평생 읽는 책

- 첫 출판 후에도 계속 돌아올 수 있다.
- 나이와 시기에 따라 다른 질문을 던질 수 있다.
- 이전의 나와 지금의 내가 같은 문장을 다르게 읽은 기록이 남는다.

### 12.2 개정되는 자기 이해

- 사용자는 자기 책의 개정판을 발행할 수 있다.
- 개정판은 과거를 지우지 않는다.
- 시간의 층을 남긴다.
- "나는 변했다"보다 "나는 나를 다르게 읽게 되었다"에 가깝다.

### 12.3 도서관 속 살아 있는 책

- 도서관의 책은 고정된 문서이면서도 독자와 대화할 수 있다.
- 다른 사람의 책을 읽을 때도 평가가 아니라 이해의 질문을 남긴다.
- 공개 대화가 아니라 개인 독서 대화가 중심이다.

### 12.4 사서의 역할

사서는 답을 주는 존재가 아니라 책을 더 잘 읽게 하는 존재다.

장기적으로 사서는 다음을 돕는다.

- 어떤 챕터를 다시 읽을지 안내한다.
- 비슷한 구조의 책을 추천한다.
- 독자가 남긴 메모를 정리한다.
- 개정판 발행 시 반영할 질문을 모은다.
- 도서관에서 지금 읽을 만한 책을 조용히 건넨다.

### 12.5 Living Book의 마지막 원칙

Living Book은 사람의 삶을 대신 말하지 않는다.

Living Book은 사람이 자기 삶을 더 오래 읽을 수 있도록 곁에 놓인 책이다.

```text
책은 대답하기 위해 살아 있는 것이 아니다.
책은 독자가 다시 질문할 수 있도록 살아 있다.
```

- 채팅 UI 구현
- DB 설계
- API 구현
- AI Prompt 작성

- 권한 시스템 구현
- 개정판 생성 로직 구현
- Reader 컴포넌트 구현
- 결제 또는 멤버십 구현
