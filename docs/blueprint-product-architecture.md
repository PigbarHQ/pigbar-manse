# Pigbar Blueprint Product Architecture

작성일: 2026-07-03

## 1. 전체 제품 개념

Blueprint는 단순 사주 리포트가 아니다.

Blueprint는 한 사람의 구조를 한 권의 책으로 출판하고, 그 책들이 모여 도서관이 되는 서비스다.

핵심 문장:

```text
한 사람은 한 권의 책이다.
사람들은 자신을 읽고, 타인을 읽고, 도서관을 만든다.
사주는 입력이고, Blueprint는 출력이다.
```

Pigbar Manse는 계산 엔진이다. Blueprint는 계산 결과를 사람이 읽을 수 있는 구조적 서사로 바꾸는 출판 시스템이다. 사용자는 "내 사주가 무엇인가"보다 "나는 어떤 구조를 가진 사람인가"를 읽는다.

Blueprint의 제품적 지향:

- 점술보다 자기 이해
- 리포트보다 책
- 피드보다 도서관
- 평가보다 독서
- 예언보다 구조
- 자극보다 오래 읽히는 문장

## 2. 전체 파이프라인

```text
Input
→ Pigbar Manse
→ Blueprint Core
→ Blueprint Writer
→ Blueprint Editor
→ Blueprint Reader
→ Blueprint Library
```

### 2.1 Input

사용자의 출생 정보를 받는다.

- 이름
- 성별
- 생년월일
- 양력/음력
- 윤달 여부
- 태어난 시간
- 태어난 지역
- 지역시 보정 옵션
- 현재 기준 시각

Input은 계산을 위한 재료일 뿐이다. 사용자는 명리학 데이터를 입력한다고 느끼기보다, 자신의 책을 만들기 위한 출판 정보를 입력한다고 느껴야 한다.

### 2.2 Pigbar Manse

역할:

- 사주 원국 계산
- 지역시 보정
- 음양력 변환
- 정밀 24절기
- 대운/세운 계산
- 구조 JSON 생성

출력:

- 원국
- 시간 보정 정보
- 음력 변환 검증 정보
- 대운/세운/월운/일진
- 십성/오행/12운성/지장간 등 구조 데이터
- 계산 warning
- debug provenance

Pigbar Manse는 해석을 하지 않는다. 정확한 계산과 근거 있는 구조 JSON을 만드는 것이 역할이다.

### 2.3 Blueprint Core

역할:

- 명리학 데이터를 사람의 구조 축으로 변환한다.
- 명리 용어를 사용자에게 직접 노출하지 않는다.
- Feature Score를 생성한다.
- 서로 모순되는 신호를 confidence와 함께 기록한다.

Core는 사주 용어를 사용자 언어로 번역하기 전의 중간 계층이다. 예를 들어 내부에서는 특정 십성, 오행, 계절성, 대운 흐름을 사용하더라도 사용자에게는 "결정 방식", "관계 경계", "회복 방식" 같은 구조 축으로 전달한다.

### 2.4 Blueprint Writer

역할:

- Core 결과를 책의 문장으로 변환한다.
- 허구를 만들지 않는다.
- 다큐멘터리 감독처럼 사실을 흥미롭게 배열한다.
- 단정 대신 관찰을 쓴다.
- 예언 대신 구조를 설명한다.

Writer는 명리학 해설자가 아니라 인물 다큐멘터리 작가에 가깝다. 계산 결과에 없는 사건, 직업, 사람, 운명을 꾸며내지 않는다.

### 2.5 Blueprint Editor

역할:

- 상투적 표현 제거
- 반복 문장 감지
- 제목 품질 관리
- 첫 문장 품질 관리
- 마지막 문장 품질 관리
- 기존 책들과 문체 중복 방지
- 과도한 위로, 예언, 단정 제거

Editor는 책을 더 멋있게 꾸미는 계층이 아니라 품질을 지키는 계층이다. 특히 "당신은 신중합니다", "타고난 리더입니다", "곧 좋은 일이 생깁니다" 같은 저품질 문장을 걸러낸다.

### 2.6 Blueprint Reader

역할:

- 책처럼 읽는 UX
- 책갈피
- 밑줄
- 메모
- 이어읽기
- 개정판
- 평생 소장

Reader는 리포트 뷰어가 아니다. 사용자가 자기 책을 여러 번 읽고, 밑줄 치고, 시간이 지난 뒤 다시 돌아오는 공간이다.

### 2.7 Blueprint Library

역할:

- 사용자 책들이 모이는 공간
- SNS가 아님
- 좋아요, 댓글, 팔로워 없음
- 책을 읽고 추천하는 구조
- 도서관/서가/전시/사서 개념

Library는 사람을 구경하는 곳이 아니라 책을 읽는 곳이다. 이름, 얼굴, 팔로워 수보다 책의 문장과 구조가 앞에 온다.

## 3. Blueprint Core v1

Blueprint Core v1은 사람을 12개 축으로 설명한다.

각 축은 다음 네 가지를 가진다.

- 질문
- 설명
- 주요 Feature
- 명리학 원자료와 연결될 수 있는 내부 source

사용자용 문장에는 식신, 정재, 편관, 장생 같은 명리 용어를 직접 쓰지 않는다. 이 용어들은 내부 feature source로만 사용한다.

### 3.1 Identity

질문:

```text
나는 누구인가?
```

설명:

Identity는 한 사람의 중심감, 자기 인식, 삶을 해석하는 기본 관점을 다룬다. 사람의 "성격"을 단정하는 축이 아니라, 자신을 세상 안에 어떻게 세우는지를 본다.

주요 Feature:

- 자기 기준의 강도
- 외부 기준에 흔들리는 정도
- 독립성
- 자기 설명 능력
- 중심을 잃을 때의 반응
- 일관성
- 자존감 회복 방식
- 정체성의 유연성

내부 source 예시:

- 일간
- 월령
- 일간의 계절 강약
- 오행 분포
- 비견/겁재 계열 신호
- 12운성의 중심성 지표

### 3.2 Thinking

질문:

```text
나는 어떻게 생각하는가?
```

설명:

Thinking은 정보를 받아들이고, 해석하고, 머릿속에서 정리하는 방식을 다룬다. 빠른 직관형인지, 구조화형인지, 반복 검토형인지, 감각 기반인지 등을 본다.

주요 Feature:

- 추상화 능력
- 패턴 인식
- 디테일 민감도
- 의심과 검증 성향
- 직관 의존도
- 학습 방식
- 정보 과부하 반응
- 관찰력
- 사고 속도

내부 source 예시:

- 인성 계열 신호
- 식상 계열 신호
- 오행 중 수/목의 배치
- 천간 투출
- 지장간의 숨은 사고 자원

### 3.3 Decision

질문:

```text
나는 어떻게 결정하는가?
```

설명:

Decision은 사람이 움직이기 전 무엇을 확인해야 하는지를 다룬다. 결정이 빠른지 느린지보다, 어떤 조건이 갖춰져야 결정을 신뢰하는지가 중요하다.

주요 Feature:

- 기준 형성 방식
- 위험 감수 성향
- 결정을 미루는 이유
- 외부 조언 수용도
- 확신 형성 속도
- 후회 민감도
- 선택 후 지속력
- 즉흥성

내부 source 예시:

- 관성/재성 균형
- 일간 강약
- 충돌 신호
- 대운 흐름의 압력
- 월주와 일주의 관계

### 3.4 Action

질문:

```text
나는 어떻게 움직이는가?
```

설명:

Action은 실제 행동 에너지와 실행 패턴을 본다. 추진력이 있는지보다, 어떤 상황에서 움직임이 살아나고 어떤 상황에서 멈추는지를 설명한다.

주요 Feature:

- 실행 속도
- 시작 에너지
- 지속력
- 루틴 적응력
- 마감 반응
- 몸으로 익히는 능력
- 압박 상황의 반응
- 실행 후 피로도

내부 source 예시:

- 식상 계열 신호
- 관성 압력
- 화/목 에너지
- 12운성의 활동성
- 대운의 활성/수축 구간

### 3.5 Relationship

질문:

```text
나는 사람을 어떻게 대하는가?
```

설명:

Relationship은 친밀감, 거리감, 신뢰 형성, 관계 안에서의 역할을 다룬다. 사람을 좋아하는지 싫어하는지가 아니라, 사람과 있을 때 어떤 구조로 관계를 만든는지를 본다.

주요 Feature:

- 친밀감 형성 속도
- 거리 조절 능력
- 의존/독립 균형
- 관계에서의 책임감
- 배려 방식
- 실망 후 반응
- 장기 관계 유지력
- 경계 설정

내부 source 예시:

- 비겁/재성/관성 관계
- 배우자궁/일지
- 지지 간 관계
- 음양 균형
- 숨은 지장간 관계성

### 3.6 Communication

질문:

```text
나는 어떻게 말하고 전달하는가?
```

설명:

Communication은 표현 방식, 설득 방식, 침묵의 이유, 말과 행동의 거리감을 본다.

주요 Feature:

- 표현 욕구
- 말의 밀도
- 직설/완곡 성향
- 설득 방식
- 감정 언어화 능력
- 침묵의 의미
- 오해 발생 패턴
- 글/말/행동 중 강한 채널

내부 source 예시:

- 식상 계열 신호
- 화/수 균형
- 천간의 노출성
- 지지의 내향성
- 충/합에 따른 표현 압력

### 3.7 Leadership

질문:

```text
나는 어떤 방식으로 영향력을 갖는가?
```

설명:

Leadership은 지배력보다 영향력의 형태를 본다. 앞에서 이끄는 사람인지, 기준을 세우는 사람인지, 분위기를 만드는 사람인지, 실무로 신뢰를 얻는 사람인지 구분한다.

주요 Feature:

- 주도성
- 책임 수용도
- 기준 제시 능력
- 사람을 움직이는 방식
- 권위와의 관계
- 조직 적응력
- 위기 시 역할
- 영향력의 지속성

내부 source 예시:

- 관성 계열 신호
- 비겁의 강도
- 재성 운용 방식
- 월주와 년주의 사회성
- 대운에서의 역할 변화

### 3.8 Conflict

질문:

```text
나는 갈등을 어떻게 겪는가?
```

설명:

Conflict는 충돌을 피하는지, 견디는지, 정면 돌파하는지, 관계를 끊는지 같은 갈등 처리 구조를 본다.

주요 Feature:

- 갈등 민감도
- 방어 방식
- 공격성 표현
- 회피/정면 대응
- 화해 가능성
- 반복되는 갈등 패턴
- 압박 상황의 언어
- 손절 기준

내부 source 예시:

- 충/형/파/해 후보 신호
- 관성 압력
- 일지와 월지 관계
- 비겁 경쟁 신호
- 오행 과다/부족

### 3.9 Growth

질문:

```text
나는 어떻게 성장하는가?
```

설명:

Growth는 사람이 어느 방식으로 성숙해지는지를 다룬다. 교육, 실패, 관계, 책임, 창작, 반복 중 무엇이 사람을 키우는지를 본다.

주요 Feature:

- 성장 트리거
- 실패에서 회복하는 방식
- 배움의 지속력
- 새로운 환경 적응력
- 자기 수정 능력
- 반복 훈련 적합도
- 성숙해지는 시기
- 대운 전환의 성장 과제

내부 source 예시:

- 인성/식상 균형
- 대운 흐름
- 12운성 변화
- 조후와 균형 신호
- 용신 후보는 v1에서는 직접 사용하지 않음

### 3.10 Wealth

질문:

```text
나는 무엇을 자원으로 바꾸는가?
```

설명:

Wealth는 돈 자체보다 자원화 능력을 본다. 무엇을 가치로 만들고, 무엇을 축적하며, 무엇을 낭비하는지 설명한다.

주요 Feature:

- 자원 감각
- 축적 방식
- 소비 패턴
- 기회 포착
- 일과 돈의 연결
- 안정성 선호
- 리스크 관리
- 가치 교환 능력

내부 source 예시:

- 재성 계열 신호
- 식상에서 재성으로 이어지는 흐름
- 관성과 재성의 균형
- 대운에서의 자원 확장/수축
- 오행 중 토/금 신호

### 3.11 Health

질문:

```text
나는 어떻게 소모되고 회복되는가?
```

설명:

Health는 의학적 진단이 아니라 에너지 관리 구조를 다룬다. 어떤 방식으로 지치고, 어떤 방식으로 회복되는지를 설명한다.

주요 Feature:

- 과로 패턴
- 회복 속도
- 스트레스가 쌓이는 위치
- 수면/리듬 민감도
- 감정 피로
- 몸과 마음의 연결감
- 쉬는 방식
- 번아웃 위험

내부 source 예시:

- 오행 과다/부족
- 조후 신호
- 수/화 균형
- 대운의 압박 구간
- 12운성의 에너지 흐름

### 3.12 Life Flow

질문:

```text
나는 어떤 흐름 위에 있는가?
```

설명:

Life Flow는 원국과 대운/세운을 연결해 현재 위치와 다음 준비를 설명한다. 예언이 아니라 구조적 흐름을 읽는 축이다.

주요 Feature:

- 현재 대운의 성격
- 전환기 여부
- 확장/정리/축적/변화 국면
- 반복되는 과제
- 준비해야 할 자원
- 관계 흐름
- 일의 흐름
- 장기 방향성

내부 source 예시:

- 대운
- 세운
- 월운
- 현재 대운과 원국의 관계
- 절기 기반 교운 시점

## 4. Blueprint Book 구조

책은 12개 질문형 챕터로 구성한다.

### Chapter 0. Prologue

목적:

- 이 사람을 한 문장으로 정의한다.
- 가장 큰 강점과 가장 큰 위험을 함께 제시한다.
- 독자가 자기 책을 계속 읽고 싶게 만든다.

필요한 Core 축:

- Identity
- Life Flow
- Growth

필요한 만세력 데이터:

- 일간
- 월령
- 오행 분포
- 대운 국면
- 강한 feature score와 위험 feature score

Writer가 생성해야 할 문장 유형:

- 한 문장 정의
- 강점과 위험의 대비
- 책 전체를 여는 첫 문장
- 독자에게 붙는 질문

금지할 표현:

- "당신은 특별한 사람입니다."
- "타고난 운명입니다."
- "반드시 성공합니다."
- 명리학 용어 직접 설명

### Chapter 1. 나는 누구인가

목적: 자기 중심과 정체성 구조를 설명한다.

필요한 Core 축: Identity, Growth

필요한 만세력 데이터: 일간, 월령, 오행 분포, 비겁 계열 source, 12운성 중심성

Writer 문장 유형:

- 자기 기준을 설명하는 문장
- 흔들릴 때의 반응
- 자신을 회복하는 방식

금지할 표현:

- "자존감이 높습니다."
- "고집이 셉니다."
- 성격 단정

### Chapter 2. 나는 어떻게 생각하는가

목적: 사고 방식과 정보 처리 방식을 설명한다.

필요한 Core 축: Thinking, Communication

필요한 만세력 데이터: 인성/식상 source, 수/목 분포, 천간 투출, 지장간

Writer 문장 유형:

- 생각이 움직이는 순서
- 이해가 깊어지는 조건
- 과부하가 오는 지점

금지할 표현:

- "머리가 좋습니다."
- "생각이 많습니다."
- 단순 지능 평가

### Chapter 3. 나는 어떻게 결정하는가

목적: 선택과 확신의 구조를 설명한다.

필요한 Core 축: Decision, Conflict, Life Flow

필요한 만세력 데이터: 관성/재성 source, 일간 강약, 대운 압력, 충돌 후보

Writer 문장 유형:

- 결정 전 필요한 기준
- 미루는 이유의 재해석
- 후회와 확신의 패턴

금지할 표현:

- "우유부단합니다."
- "결단력이 있습니다."
- 선택을 잘한다/못한다 식 평가

### Chapter 4. 나는 사람을 어떻게 대하는가

필요한 Core 축: Relationship, Communication

필요한 만세력 데이터: 일지, 배우자궁, 비겁/재성/관성 source, 지지 관계

Writer 문장 유형:

- 친밀감 형성 방식
- 관계에서 지키는 선
- 실망 후 반응

금지할 표현:

- "인복이 있습니다."
- "사람을 좋아합니다."
- 관계 운 단정

### Chapter 5. 사람들은 나를 어떻게 보는가

목적: 외부에 비치는 인상과 실제 구조의 차이를 설명한다.

필요한 Core 축: Leadership, Communication, Identity

필요한 만세력 데이터: 년주, 월주, 천간 노출성, 사회적 source

- 첫인상과 실제의 차이

금지할 표현:

- "카리스마가 있습니다."

- 강점이 켜지는 조건

금지할 표현:

### Appendix. My Notes

목적:

- 독자가 직접 자기 문장을 남긴다.
- 밑줄과 메모를 모은다.
- 시간이 지난 뒤 개정판을 위한 재료가 된다.

필요한 만세력 데이터: 없음.

Writer 문장 유형:

- 빈 공간을 여는 질문
- 독자의 결론을 요청하는 문장

금지할 표현:

- 정답 제공
- 강요형 회고 질문

## 5. Blueprint Writer 원칙

Writer는 다음 구조를 따른다.

```text
Hook
→ Story
→ Mirror
→ Twist
→ Expansion
→ Ending
```

### Hook

첫 문장이 강해야 한다. 독자가 "이건 내 이야기일 수 있다"고 느껴야 한다.

나쁜 예:

```text
당신은 신중한 사람입니다.
```

좋은 예:

```text
당신은 결정을 미루는 사람이 아니라, 기준이 충분히 설 때까지 움직이지 않는 사람입니다.
```

### Story

허구의 사건을 만들지 않는다. 구조가 일상에서 어떻게 보일 수 있는지 관찰 가능한 장면으로 배열한다.

### Mirror

독자가 자신을 비춰볼 수 있게 한다. 판단하지 않고 비춘다.

### Twist

상투적 해석을 한 번 비튼다. 약점처럼 보이는 구조가 어떤 조건에서 강점이 되는지 보여준다.

### Expansion

개인의 구조를 관계, 일, 성장, 시간의 흐름으로 확장한다.

### Ending

마지막 문장은 다음 장을 읽게 하거나, 장 전체를 다시 생각하게 만들어야 한다.

Writer 규칙:

- 첫 문장이 강해야 한다.
- 각 챕터 제목 자체가 읽고 싶어야 한다.
- 허구를 만들지 않는다.
- 위로나 예언처럼 쓰지 않는다.
- 사람을 단정하지 않는다.
- 명리학 용어를 직접 설명하지 않는다.
- "당신은 신중합니다" 같은 평범한 표현을 피한다.
- 구조를 정확히 말하되 문장에는 온도를 둔다.

## 6. Blueprint Reader 원칙

Reader는 조용한 앱이어야 한다.

원칙:

- 광고 없음
- 좋아요 없음
- 댓글 없음
- 팔로워 없음
- 알림 숫자 없음
- 책처럼 읽기
- 이어읽기
- 책갈피
- 밑줄
- 개인 메모
- 나만의 결론 작성
- 개정판 관리
- 평생 소장

Reader의 핵심 경험:

```text
내 책을 조용히 읽는다.
필요한 문장에 밑줄을 긋는다.
시간이 지난 뒤 다시 돌아온다.
내가 직접 내 결론을 쓴다.
```

## 7. Blueprint Library 원칙

Library는 SNS가 아니다.

원칙:

- 사람을 팔로우하지 않는다.
- 책을 읽는다.
- 사람을 평가하지 않는다.
- 책을 추천한다.
- 누군지 몰라도 읽을 수 있다.
- 익명성이 기본이다.
- 이름/얼굴/팔로워 수를 앞세우지 않는다.

도서관 성장 단계:

| 규모 | 이름 | 의미 |
|---:|---|---|
| 10권 | 작은 책장 | 초기 사용자들의 개인 서가 |
| 100권 | 작은 도서관 | 큐레이션이 시작되는 단계 |
| 1,000권 | 커뮤니티 도서관 | 주제별 전시 가능 |
| 10,000권 | Blueprint Library | 검색과 추천이 핵심 기능이 됨 |
| 100,000권 | World Blueprint Library | 사람 구조의 거대한 익명 아카이브 |

Library 기능 후보:

- 오늘의 한 권
- 이번 달 전시
- 사서 추천
- 구조 검색
- 지역 도서관
- 개관 컬렉션
- Founding Collection
- 개정판 서가

## 8. BM 초안

초기 BM:

```text
내 책 생성: 9,900원
1개월 도서관 이용권 포함
이후 월 2,900원 또는 연 29,000원
```

원칙:

- 내 책은 평생 소장
- 도서관은 멤버십
- 책값을 내는 것이 아니라 도서관 회원이 되는 경험
- 구독을 강요하지 않음
- 결제보다 소장 경험이 먼저 와야 함

추가 BM 후보:

- 가족판
- 커플판
- 개정판
- 실물 양장본
- 선물하기
- 기업/팀 Blueprint

## 10. MVP 범위

MVP는 반드시 작게 잡는다.

MVP:

1. Pigbar Manse 결과 JSON 입력
2. Blueprint Core 축별 Feature JSON 생성
3. 1권의 Blueprint Book 초안 생성
4. Reader에서 책처럼 보여주기
5. 내 서재에 저장

MVP에서 중요한 것:

- 책 한 권의 품질
- 첫 문장과 챕터 제목의 품질
- Core feature와 Writer 문장의 추적 가능성
- Reader의 조용한 독서 경험
- 저장과 이어읽기

## 13. 개발 전에 결정해야 할 질문

- Core feature score는 0-100 점수인가, low/medium/high 등급인가?
- 각 feature의 confidence를 어떻게 표시할 것인가?
- 명리학 source를 사용자에게 완전히 숨길 것인가, debug로만 열 것인가?
- 책의 기본 분량은 어느 정도인가?
- Prologue를 먼저 보여줄 것인가, 책 표지를 먼저 보여줄 것인가?
- 책 제목은 사용자 이름 기반인가, 구조 기반인가?
- Reader의 밑줄/메모는 로컬 우선인가, 계정 동기화인가?
- 개정판은 언제 생성되는가?
- Library 공개 시 익명성은 어떻게 보장하는가?
- "내 책"의 소유권과 삭제 권한을 어떻게 정의하는가?
